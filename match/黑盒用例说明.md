# 公开黑盒用例说明

公开黑盒 API 用例代码位于：

```text
test-cases/
```

API 基线见：

```text
API基线文档.md
```

## 运行方式

```bash
mvn -f code/pom.xml install
mvn -f test-cases/pom.xml test
```

`test-cases/` 是独立 Maven 测试项目，不在 `code/pom.xml` 的 reactor 模块列表中。第一条命令把当前业务代码安装到本地 Maven 仓库，第二条命令使用该构建结果启动完整 ShopHub 应用并执行 REST 黑盒测试。

运行单个公开测试类：

```bash
mvn -f test-cases/pom.xml -Dtest=PublicL1Test test
```

不需要修改任何 POM 来运行公开黑盒用例。

## 用例说明

公开用例基于设计文档和 API 基线编写，用于帮助参赛者验证关键 REST 场景。公开用例不覆盖全部验收范围，通过公开用例不代表实现已完整符合设计。

## Given-When-Then 规格

### 0. 通用执行约束

每个公开用例必须独立运行，并且只通过 REST API 创建前置条件和观察结果。

每个用例统一执行：

```text
BeforeEach:
1. 测试 harness 启动新的 Spring Boot 上下文
2. 使用随机 H2 内存库和测试 profile
3. 测试 harness seed 管理员账号
4. 生成 testRunId
5. 通过 POST /api/v1/users/login 获取 adminToken

AfterEach:
1. 测试 harness 关闭当前 Spring Boot 上下文
2. 丢弃随机 H2 内存库、缓存和测试静态状态
```

所有唯一字段带 `testRunId`，例如：

```text
email=user-{testRunId}@example.com
skuCode=SKU-{testRunId}
warehouseCode=WH-{testRunId}
couponCode=CP-{testRunId}
externalOrderNo=EO-{testRunId}
clientPaymentNo=PAY-{testRunId}
```

## 1. 预期通过的基础链路

### PUB-001 用户注册激活登录

```text
Given:
1. 环境已由测试 harness 初始化，管理员已 seed

When:
1. POST /api/v1/users/register，使用唯一 email/phone
2. POST /api/v1/users/activate，使用注册/通知返回的 activationToken
3. POST /api/v1/users/login
4. GET /api/v1/users/me，携带 userToken

Then:
1. 注册返回 HTTP 201，status=PENDING_ACTIVATION
2. 激活返回 HTTP 200，status=ACTIVE
3. 登录返回 HTTP 200，body.token 非空
4. me 返回当前用户 email 和 userId

Cleanup:
1. 测试方法结束后由 harness 丢弃当前上下文
```
