# 积分与会员服务设计

## 1. 模块职责

积分与会员服务负责积分赚取、积分抵扣、积分冻结、积分过期、会员等级、等级倍率和会员权益。

## 2. 积分赚取

订单支付成功后发放积分：

```text
订单积分 = 订单实付金额 × 会员等级倍率 × 活动系数
```

活动系数默认为 1.0，大促期间由运营配置。

## 3. 积分抵扣

单笔订单积分抵扣规则：

1. 最多可使用 10,000 积分，等值 100 元。
2. 抵扣金额不超过订单实付金额的 50%。
3. 积分兑换比例：100 积分 = 1 元。

计算公式：

```text
按比例可用积分 = 订单金额 × 100 × 0.5
实际可用积分 = min(用户可用积分, 10000, 按比例可用积分)
抵扣金额 = 实际可用积分 / 100
```

示例 1：用户有 20,000 积分，订单金额 300 元。

```text
按 50% 比例上限：300 × 100 × 0.5 = 15,000 积分
取最小值：min(20,000, 15,000, 10,000) = 10,000 积分
抵扣金额：100 元
```

示例 2：用户有 5,000 积分，订单金额 80 元。

```text
按 50% 比例上限：80 × 100 × 0.5 = 4,000 积分
取最小值：min(5,000, 4,000, 10,000) = 4,000 积分
抵扣金额：40 元
```

## 4. 积分过期

积分有效期为 12 个自然月。每月 1 号凌晨系统批量扫描过期积分，将过期积分扣减并记录日志。

积分抵扣时不得使用已过期积分。

## 5. 会员等级

| 等级 | 条件 | 积分倍率 |
|------|------|----------|
| NORMAL | 默认 | 1.0 |
| SILVER | 年消费满 1,000 | 1.1 |
| GOLD | 年消费满 5,000 | 1.2 |
| PLATINUM | 年消费满 20,000 | 1.5 |

会员等级统计需要订单累计消费数据时，积分与会员服务不得直接访问订单表或订单模块 Repository。必须通过 `OrderQueryService`、订单销售统计接口或公开的本地查询契约获取累计消费金额。

## 6. REST API

| Method | URL | 说明 |
|--------|-----|------|
| GET | `/api/v1/loyalty/points` | 查询积分 |
| POST | `/api/v1/loyalty/points/estimate-redeem` | 估算积分抵扣 |
| GET | `/api/v1/loyalty/points/history` | 积分流水 |
| GET | `/api/v1/loyalty/member-level` | 查询会员等级 |
| POST | `/api/v1/admin/loyalty/points/expire` | 处理过期积分 |
