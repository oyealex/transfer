# 购物车服务设计

## 1. 模块职责

购物车服务负责购物车增删改查、商品有效性预校验、库存摘要展示、价格预估和购物车 TTL 管理。

购物车是临时数据，必须存储在本地 Caffeine Cache 中，Key 为：

```text
cart:{userId}
```

TTL 为 7 天。数据库只保存已提交订单，不保存临时购物车。

## 2. 购物车规则

| 规则 | 要求 |
|------|------|
| 最大商品种类 | 100 |
| 单项数量 | 1 到 999 |
| SKU 状态 | 必须为 ON_SHELF |
| 库存展示 | 通过 InventoryQueryService 查询 |
| 价格预估 | 通过 PromotionCalculationService 计算 |

同一个 SKU 重复加入购物车时，数量累加。

## 3. 价格预估

购物车价格预估返回：

1. 商品原价合计。
2. 满减优惠。
3. 优惠券可用列表。
4. 会员折扣。
5. 预计运费。
6. 预计应付金额。

预估结果不锁定价格，最终价格以下单时重新计算为准。

## 4. REST API

| Method | URL | 说明 |
|--------|-----|------|
| POST | `/api/v1/cart/items` | 添加购物车 |
| GET | `/api/v1/cart` | 查询购物车 |
| PUT | `/api/v1/cart/items/{skuId}` | 修改数量 |
| DELETE | `/api/v1/cart/items/{skuId}` | 删除商品 |
| DELETE | `/api/v1/cart` | 清空购物车 |
| POST | `/api/v1/cart/estimate` | 价格预估 |
