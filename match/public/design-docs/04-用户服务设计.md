# 用户服务设计

## 1. 模块职责

用户服务负责用户注册、邮箱激活、登录认证、JWT 签发、用户资料、地址管理、账户冻结、角色权限和购买者身份维护。

用户服务不负责订单、支付、积分和评价业务，只提供查询用户状态和认证信息的能力。

## 2. 领域模型

| 实体 | 说明 |
|------|------|
| User | 用户账号，包含邮箱、手机号、密码哈希、状态、角色 |
| UserProfile | 用户资料，包含昵称、头像、生日、性别 |
| UserAddress | 收货地址，包含省、市、区、详细地址、收货人、手机号 |
| EmailActivationToken | 邮箱激活令牌 |
| LoginSession | 登录会话记录 |

用户状态：

| 状态 | 说明 |
|------|------|
| PENDING_ACTIVATION | 已注册，待邮箱激活 |
| ACTIVE | 正常可登录、可下单 |
| FROZEN | 已冻结，不可登录、不可下单 |
| CLOSED | 已注销 |

## 3. 注册流程

注册流程必须包含邮箱激活：

```text
提交注册信息
→ 校验邮箱和手机号唯一
→ 保存用户，状态为 PENDING_ACTIVATION
→ 生成邮箱激活令牌
→ 通过 LocalNotificationService 发送激活邮件
→ 用户点击激活链接
→ 用户状态变更为 ACTIVE
→ 注册完成，可登录
```

未激活用户不得登录，也不得创建订单。

## 4. 登录流程

1. 校验用户名或邮箱存在。
2. 校验用户状态为 ACTIVE。
3. 校验密码哈希。
4. 生成 JWT。
5. 记录登录会话。

冻结用户登录返回 403，错误码为 `USER_FROZEN`。

## 5. 地址格式化

地址格式化方法签名为：

```java
format(String province, String city, String district, String detail)
```

输出格式：

```text
省 + 市 + 区 + 详细地址
```

参数顺序不得调整。

## 6. 查询接口

`UserQueryService` 提供：

| 方法 | 说明 |
|------|------|
| `getUserById(Long userId)` | 查询用户基础信息 |
| `isActive(Long userId)` | 用户是否正常 |
| `isFrozen(Long userId)` | 用户是否冻结 |
| `getDefaultAddress(Long userId)` | 查询默认收货地址 |

订单创建前必须校验用户存在且 `isFrozen(userId) == false`。

## 7. REST API

| Method | URL | 说明 |
|--------|-----|------|
| POST | `/api/v1/users/register` | 注册用户 |
| POST | `/api/v1/users/activate` | 激活邮箱 |
| POST | `/api/v1/users/login` | 登录 |
| GET | `/api/v1/users/me` | 当前用户信息 |
| POST | `/api/v1/users/addresses` | 新增地址 |
| GET | `/api/v1/users/addresses` | 地址列表 |
| PUT | `/api/v1/users/addresses/{addressId}` | 修改地址 |
| DELETE | `/api/v1/users/addresses/{addressId}` | 删除地址 |
| POST | `/api/v1/admin/users/{userId}/freeze` | 冻结用户 |
| POST | `/api/v1/admin/users/{userId}/unfreeze` | 解冻用户 |
