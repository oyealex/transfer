# 商品服务设计

## 1. 模块职责

商品服务负责 SPU、SKU、品牌、类目、属性模板、商品上下架、商品搜索和商品详情聚合。商品服务不拥有库存数量，商品详情展示库存状态时必须通过 `InventoryQueryService` 查询库存摘要。

## 2. 领域模型

| 实体 | 说明 |
|------|------|
| ProductSpu | 标准商品单元，描述商品主体 |
| ProductSku | 库存量单位，包含规格、售价、状态 |
| Category | 类目树 |
| Brand | 品牌 |
| AttributeTemplate | 类目属性模板 |
| ProductTag | 商品标签 |

SKU 状态：

| 状态 | 说明 |
|------|------|
| DRAFT | 草稿 |
| ON_SHELF | 已上架，可销售 |
| OFF_SHELF | 已下架 |
| DELETED | 已删除 |

## 3. 商品详情

商品详情返回：

1. SPU 基础信息。
2. SKU 列表。
3. 类目和品牌信息。
4. 属性和规格。
5. 价格。
6. 库存摘要。

库存摘要必须通过 `InventoryQueryService.getStockSummary(skuId)` 获取，不得直接访问库存表或注入库存模块 Repository。

## 4. 商品搜索

搜索支持：

| 条件 | 说明 |
|------|------|
| keyword | 商品名称、卖点模糊匹配 |
| categoryId | 类目过滤，包含子类目 |
| brandId | 品牌过滤 |
| minPrice/maxPrice | 价格区间 |
| tags | 标签过滤 |
| onlyOnShelf | 是否只查已上架 |

默认只展示 `ON_SHELF` 商品。

## 5. ProductQueryService

| 方法 | 说明 |
|------|------|
| `getSku(Long skuId)` | 查询 SKU |
| `getSkuForSale(Long skuId)` | 查询可销售 SKU，不可销售时抛错 |
| `listSkuByIds(Collection<Long> skuIds)` | 批量查询 SKU |
| `getProductSnapshot(Long skuId)` | 返回订单商品快照 |

库存服务获取商品名称、SKU 编码、规格时必须使用 `ProductQueryService`，不得在库存模块内重复定义 Product JPA Entity。

## 6. REST API

| Method | URL | 说明 |
|--------|-----|------|
| POST | `/api/v1/admin/products/spu` | 创建 SPU |
| POST | `/api/v1/admin/products/sku` | 创建 SKU |
| POST | `/api/v1/admin/products/sku/{skuId}/on-shelf` | SKU 上架 |
| POST | `/api/v1/admin/products/sku/{skuId}/off-shelf` | SKU 下架 |
| GET | `/api/v1/products` | 商品列表 |
| GET | `/api/v1/products/search` | 商品搜索 |
| GET | `/api/v1/products/{skuId}` | 商品详情 |
| GET | `/api/v1/categories/tree` | 类目树 |
