# 评价服务设计

## 1. 模块职责

评价服务负责商品评价、追评、图片评价、评价审核、敏感词过滤、评价展示和评价奖励积分。

评价服务必须调用订单服务验证购买记录，不得允许未购买用户评价。

## 2. 评价前提

用户必须满足以下条件才可发表评价：

1. 用户已登录。
2. 用户通过订单服务验证已购买该商品。
3. 订单状态为 DELIVERED 或 COMPLETED。
4. 同一订单明细只能发表一次主评价。

验证接口：

```text
GET /api/v1/orders/verify-purchase?userId=&productId=
```

## 3. 审核流程

```text
用户提交评价
→ 敏感词过滤
→ 状态为 PENDING_REVIEW
→ 管理员审核
→ 审核通过后展示
→ 发布 ReviewApprovedEvent
→ 积分服务发放评价奖励
```

敏感词过滤采用包含匹配：只要评价内容包含任一敏感词，即不得直接进入 APPROVED，应进入 PENDING_REVIEW 或 REJECTED。不得只做完全相等匹配。

## 4. 评价状态

| 状态 | 说明 |
|------|------|
| PENDING_REVIEW | 待审核 |
| APPROVED | 审核通过 |
| REJECTED | 审核拒绝 |
| HIDDEN | 已隐藏 |

## 5. REST API

| Method | URL | 说明 |
|--------|-----|------|
| POST | `/api/v1/reviews` | 发布评价 |
| POST | `/api/v1/reviews/{reviewId}/append` | 追评 |
| GET | `/api/v1/reviews/product/{productId}` | 商品评价列表 |
| GET | `/api/v1/reviews/my` | 我的评价 |
| POST | `/api/v1/admin/reviews/{reviewId}/approve` | 审核通过 |
| POST | `/api/v1/admin/reviews/{reviewId}/reject` | 审核拒绝 |
