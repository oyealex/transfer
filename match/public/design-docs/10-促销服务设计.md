# 促销服务设计

## 1. 模块职责

促销服务负责优惠券、满减活动、秒杀活动、阶梯定价、会员折扣和优惠叠加计算。

促销服务只负责计算优惠和校验优惠适用性，不创建订单、不扣减库存、不处理支付。

## 2. 优惠券规则

优惠券类型：

| 类型 | 说明 |
|------|------|
| DISCOUNT | 折扣券，如 8 折 |
| AMOUNT_OFF | 立减券 |
| THRESHOLD_OFF | 满减券 |

折扣券计算规则：

```text
8 折券将原价乘以 0.8，即 price × 0.8
优惠金额 = price × (1 - 0.8)
```

优惠券校验顺序：

```text
1. 优惠券是否存在
2. 是否在有效期内
3. 是否满足使用门槛
4. 是否适用于当前商品
5. 用户是否满足使用限制
6. 是否已使用
```

## 3. 优惠叠加顺序

优惠叠加顺序固定为：

```text
满减活动 → 优惠券折扣 → 会员专属折扣
```

后一步基于前一步的结果计算。

示例：

```text
商品金额 300
满减 -30 后为 270
8 折优惠券后为 216
会员 95 折后为 205.20
```

## 4. 秒杀规则

秒杀订单必须校验：

1. 活动处于进行中。
2. SKU 参与活动。
3. 用户未超过限购数量。
4. 秒杀库存充足。
5. 秒杀价格不参与普通满减。

## 5. REST API

| Method | URL | 说明 |
|--------|-----|------|
| POST | `/api/v1/admin/promotions/coupons` | 创建优惠券 |
| POST | `/api/v1/promotions/coupons/claim` | 领取优惠券 |
| GET | `/api/v1/promotions/coupons/my` | 我的优惠券 |
| POST | `/api/v1/promotions/calculate` | 计算优惠 |
| POST | `/api/v1/admin/promotions/full-reductions` | 创建满减活动 |
| POST | `/api/v1/admin/promotions/seckill` | 创建秒杀活动 |
