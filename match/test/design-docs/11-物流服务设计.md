# 物流服务设计

## 1. 模块职责

物流服务负责发货单、拣货单、物流面单、扫码出库、物流轨迹、签收、运费模板和物流状态回调。

订单支付成功后，物流服务通过监听 `OrderPaidEvent` 创建发货单，不应由订单服务同步调用。

## 2. 发货流程

```text
支付确认
→ 生成拣货单
→ 仓库人员拣货
→ 打印物流面单
→ 扫码出库
→ 物流揽收
→ 更新物流状态
```

不得跳过拣货单和面单步骤直接出库。

## 3. 物流状态

| 状态 | 说明 |
|------|------|
| CREATED | 发货单已创建 |
| PICKING | 拣货中 |
| LABEL_PRINTED | 面单已打印 |
| OUTBOUND | 已出库 |
| COLLECTED | 物流揽收 |
| IN_TRANSIT | 运输中 |
| DELIVERED | 已签收 |
| EXCEPTION | 物流异常 |

物流状态变更后，必须通过 `OrderLogisticsStatusUpdater` 更新对应订单的物流状态。

## 4. 运费规则

默认运费为 8 元。订单商品金额满 199 元免运费。

运费模板可按省份、重量和商品件数配置。运费最终以订单创建时计算结果为准。

## 5. REST API

| Method | URL | 说明 |
|--------|-----|------|
| GET | `/api/v1/logistics/order/{orderId}` | 查询订单物流 |
| POST | `/api/v1/admin/logistics/shipments/{shipmentId}/pick` | 生成拣货单 |
| POST | `/api/v1/admin/logistics/shipments/{shipmentId}/print-label` | 打印面单 |
| POST | `/api/v1/admin/logistics/shipments/{shipmentId}/outbound` | 扫码出库 |
| POST | `/api/v1/logistics/callback` | 物流状态回调 |
| POST | `/api/v1/admin/logistics/freight-templates` | 创建运费模板 |
