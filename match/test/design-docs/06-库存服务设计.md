# 库存服务设计

## 1. 模块职责

库存服务负责仓库、入库、出库、库存查询、库存预占、库存释放、支付后扣减、盘点、调拨和低库存预警。

库存服务不维护商品主数据。需要商品名称、SKU 编码和规格时，通过 `ProductQueryService` 获取。

## 2. 库存模型

| 实体 | 说明 |
|------|------|
| Warehouse | 仓库 |
| InventoryStock | SKU 在仓库中的库存 |
| StockReservation | 库存预占记录 |
| InboundOrder | 入库单 |
| OutboundOrder | 出库单 |
| StockAdjustment | 盘点调整 |
| StockWarningRule | 低库存预警规则 |

库存字段：

```text
availableStock = onHandStock - reservedStock
```

当 `availableStock >= requestQuantity` 时库存充足。

## 3. 下单库存处理

创建订单时只预占库存：

```text
校验库存可用
→ 创建 StockReservation
→ 增加 reservedStock
→ 不减少 onHandStock
```

支付成功后扣减库存：

```text
接收 OrderPaidEvent 或支付确认命令
→ 查找预占记录
→ 减少 onHandStock
→ 减少 reservedStock
→ 生成 OutboundOrder
```

订单取消或超时未支付时释放预占库存：

```text
减少 reservedStock
→ 关闭 StockReservation
```

## 4. 多仓分配

库存分配优先级：

1. 用户默认地址所在省份匹配仓库服务区域。
2. 可用库存充足。
3. 距离优先。
4. 仓库优先级。

单个 SKU 可拆分到多个仓库，但同一订单的同一 SKU 优先单仓发出。

## 5. InventoryQueryService

| 方法 | 说明 |
|------|------|
| `getStockSummary(Long skuId)` | 查询 SKU 库存摘要 |
| `checkAvailability(Long skuId, int quantity)` | 校验可用库存 |
| `listAvailableWarehouses(Long skuId)` | 查询可用仓库 |

## 6. InventoryReservationService

| 方法 | 说明 |
|------|------|
| `reserve(orderId, items)` | 创建订单时预占库存 |
| `release(orderId)` | 取消或超时释放库存 |
| `deductAfterPayment(orderId)` | 支付成功后扣减库存 |

## 7. REST API

| Method | URL | 说明 |
|--------|-----|------|
| POST | `/api/v1/admin/warehouses` | 创建仓库 |
| POST | `/api/v1/admin/inventory/inbound` | 入库 |
| POST | `/api/v1/admin/inventory/outbound` | 手工出库 |
| GET | `/api/v1/inventory/sku/{skuId}` | 查询 SKU 库存 |
| POST | `/api/v1/inventory/check` | 库存可用性校验 |
| POST | `/api/v1/admin/inventory/adjustments` | 盘点调整 |
| GET | `/api/v1/admin/inventory/warnings` | 低库存预警 |
