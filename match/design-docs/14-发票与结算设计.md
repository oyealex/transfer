# 发票与结算设计

## 1. 模块职责

发票与结算作为 payment-service 子域实现，负责发票申请、部分开票、税额计算、发票状态、结算批次和销售对账。

## 2. 发票类型

| 类型 | 说明 |
|------|------|
| PERSONAL | 个人普通发票 |
| COMPANY | 企业普通发票 |
| VAT_SPECIAL | 增值税专用发票 |

## 3. 部分开票

一个订单可以申请多张发票。单张发票金额不得超过订单剩余可开票金额。

```text
剩余可开票金额 = 订单实付金额 - 已开票成功金额
```

如果订单实付金额为 800 元，已开票 300 元，则剩余可开票金额为 500 元。

## 4. 税率

税率从配置 `invoice.tax-rate` 读取，默认 6%。

```text
税额 = 发票金额 × 税率
价税合计 = 发票金额
不含税金额 = 发票金额 - 税额
```

税额按 `RoundingMode.HALF_UP` 保留两位小数。

## 5. 结算批次

结算批次按自然日生成，包含支付成功且未结算的订单、退款和发票数据。

结算批次生成后不可修改，只能作废后重建。
