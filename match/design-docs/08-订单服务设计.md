# 订单服务设计

## 1. 模块职责

订单服务负责订单创建、订单明细、价格确认、库存预占、订单状态机、拆单、批量导入、取消审核、订单查询和销售统计。

订单服务不直接处理支付资金，不直接创建物流单，不直接发放积分。支付、物流、积分通过对应模块接口或事件协作。

## 2. 订单状态

| 状态 | 说明 |
|------|------|
| CREATED | 已创建，待支付 |
| PAYING | 支付中 |
| PAID | 已支付 |
| PICKING | 拣货中 |
| SHIPPED | 已发货 |
| DELIVERED | 已签收 |
| COMPLETED | 已完成 |
| CANCEL_REVIEWING | 取消审核中 |
| CANCELLED | 已取消 |
| REFUNDING | 退款中 |
| REFUNDED | 已退款 |
| CLOSED | 已关闭 |

## 3. 下单完整流程

```text
1. 校验用户存在且未被冻结
2. 校验商品存在、已上架、价格有效
3. 校验购物车项或直接购买项
4. 校验库存可用并预占库存
5. 执行风控校验，高风险订单拒绝
6. 计算优惠：满减 → 优惠券 → 会员折扣
7. 计算积分抵扣
8. 计算运费和包装费
9. 创建订单和订单明细
10. 发布 OrderCreatedEvent
11. 返回订单，等待支付
```

## 4. 订单计价公式

```text
商品总金额 = Σ(商品单价 × 数量)
优惠抵扣金额 = 满减优惠 + 优惠券优惠 + 会员折扣优惠
订单总价 = 商品总金额 + 运费 + 包装费 - 优惠抵扣金额 - 积分抵扣金额
```

订单总价不得小于 0.01。

所有金额最终使用 `RoundingMode.HALF_UP` 四舍五入到分。

## 5. 订单超时

用户提交订单后，若在 60 分钟内未完成支付，系统自动取消订单并释放预占库存。

## 6. 订单取消规则

| 当前状态 | 取消规则 |
|----------|----------|
| CREATED | 用户可直接取消，释放库存 |
| PAYING | 用户可取消，支付回调到达后按幂等规则处理 |
| PAID | 进入商家取消审核，审核通过后按退款流程处理，扣除手续费 |
| SHIPPED | 不可取消，只能申请售后 |
| DELIVERED | 不可取消，可申请退货退款 |

已支付订单不得直接跳转到 `CANCELLED`，必须先进入 `CANCEL_REVIEWING`。

## 7. 批量订单导入

批量导入订单时逐条校验，合法订单逐条创建，非法订单记录失败原因并跳过。最终返回每条的创建结果明细。

任何一条失败不得导致整批订单回滚。

## 8. OrderQueryService

| 方法 | 说明 |
|------|------|
| `getOrder(Long orderId)` | 查询订单 |
| `getPayableOrder(Long orderId)` | 查询可支付订单 |
| `verifyPurchase(userId, productId)` | 验证用户是否购买并签收商品 |
| `getOrderAmount(Long orderId)` | 查询应付金额 |

支付服务必须通过 `OrderQueryService` 查询订单信息，不得直接访问订单表。

## 9. REST API

| Method | URL | 说明 |
|--------|-----|------|
| POST | `/api/v1/orders/create` | 创建订单，成功返回 201 Created |
| GET | `/api/v1/orders/{orderId}` | 订单详情 |
| GET | `/api/v1/orders` | 我的订单列表 |
| POST | `/api/v1/orders/{orderId}/cancel` | 取消订单 |
| POST | `/api/v1/admin/orders/{orderId}/cancel-review` | 取消审核 |
| POST | `/api/v1/orders/batch` | 批量创建订单 |
| GET | `/api/v1/orders/verify-purchase` | 验证购买记录 |
| GET | `/api/v1/admin/orders/statistics/sales` | 销售统计 |
