# 支付服务设计

## 1. 模块职责

支付服务负责支付单、支付发起、支付回调、支付确认、退款单、退款计算、对账、发票和结算子域。

支付服务通过 `OrderQueryService` 查询订单信息，通过 `OrderPaymentStatusUpdater` 更新订单支付状态，不得直接查询或更新订单数据库表。

## 2. 支付规则

本系统仅支持全额支付。支付金额必须等于订单应付金额。

支付校验：

1. 订单存在。
2. 订单状态可支付。
3. 支付金额等于订单应付金额。
4. 支付方式受支持。
5. 支付单未重复成功。

支付金额小于或大于订单应付金额均应拒绝。

## 3. 支付成功流程

```text
接收支付成功回调
→ 校验签名和幂等键
→ 更新支付单为 SUCCESS
→ 更新订单支付状态为 PAID
→ 触发库存支付后扣减
→ 发布 PaymentSucceededEvent 和 OrderPaidEvent
→ 返回回调成功
```

物流创建、积分发放和通知发送必须通过本地事件监听器异步触发。任一后置动作失败不得导致支付确认失败。

## 4. 退款流程

```text
用户申请退款
→ 商家审核退货
→ 买家寄回商品
→ 仓库验收，确认商品完好
→ 财务退款
→ 发布 RefundCompletedEvent
→ 通知用户
```

商家审核通过后不得直接退款，必须等待仓库验收。

## 5. 退款金额

退款金额公式：

```text
退款金额 = 实付金额 × (1 - 手续费率)
```

默认手续费率为 2%，即：

```text
refund = paidAmount × 0.98
```

不得额外扣除固定费用。

## 6. 发票规则

发票税率从本地配置 `invoice.tax-rate` 读取，默认值 6%。

发票支持部分开票：

1. 单张发票金额不得超过订单剩余可开票金额。
2. 一个订单可开多张发票。
3. 累计开票金额不得超过订单实付金额。
4. 发票税额 = 发票金额 × 税率。

## 7. REST API

| Method | URL | 说明 |
|--------|-----|------|
| POST | `/api/v1/payment/pay` | 发起支付 |
| POST | `/api/v1/payment/callback` | 支付回调 |
| GET | `/api/v1/payment/{paymentNo}` | 查询支付单 |
| POST | `/api/v1/refunds/apply` | 申请退款 |
| POST | `/api/v1/admin/refunds/{refundId}/review` | 商家审核退款 |
| POST | `/api/v1/admin/refunds/{refundId}/warehouse-accept` | 仓库验收 |
| GET | `/api/v1/refunds/{refundId}` | 查询退款 |
| POST | `/api/v1/invoices` | 申请发票 |
| GET | `/api/v1/invoices/order/{orderId}` | 查询订单发票 |
| POST | `/api/v1/admin/settlements/batches` | 生成结算批次 |
